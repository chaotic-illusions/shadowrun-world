>> LOADOUT CANVAS CONCEPTS

Throwaway prototype · three ways to plan a deck loadout without endless scrolling. Not wired to the live builder.
Storage = everything resident on the deck (the kit you bring). Active = the working set running now (cap = MPCP × 100). Active programs also occupy storage (active ⊂ storage). Reserves staged in storage are what you can Swap Memory into Active during a run — so picking them is still a loadout-build decision, even though the swap itself happens live.

Click tiles / chips to move programs between Library → Storage → Active.
One board is storage. The glowing frame inside it is active — because active is drawn inside storage, the “active also eats storage” rule needs no explanation. Go live lifts a reserve into the frame; stow drops it back. Tiles are sized by Mp, so a fat program (Black Hammer!) visibly hogs the board.

Add a program to storage first; then go live. Grouped by type so variants collapse.
Variants collapse to one card per type. The starter is the variant you run; the bench below it is the same type’s other copies plus alternates you own. Picking a bench item here means “stage this in storage so I can swap to it mid-run.” Tap a bench chip to promote it to starter (the old starter drops to the bench).
Active is a fixed dock (a persona hotbar) — always the same height, never scrolls. Storage + library live in the tray drawer beneath. Click a tray chip to promote it to the dock; click a dock slot to send it back down. One-shots stack with ×N.
